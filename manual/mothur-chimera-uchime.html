Remove chimeric sequences

Description

Given a fasta-formatted alignment file, removes chimeric sequences.

Parameters

  • Method (vsearch, uchime) [vsearch]
  • Reference (bacterial, full, none) [bacterial]
  • Dereplicate (false, true) [false]

Details

Removes chimeric sequences (artifacts which contain parts of several sequences) from an alignment using the vsearch or uchime method. The vsearch method is faster.

You can select de novo or reference based chimera detection. As a reference you can use the bacterial subset of the 16S rRNA Silva gold reference set. For the uchime method also the full 16S rRNA Silva gold reference set is available. When using the de novo approach, the tool checks for chimeras using the more abundant sequences in your data as the reference.

The dereplicate parameter is by default false, so when a sequence is detected as chimeric in one sample it is assigned chimeric all the samples. If you want to limit the assignment to the original sample, set the dereplicate parameter to true.

This tool is based on the Chimera.vsearch, Chimera.uchime, and Remove.seqs commands of the Mothur package.

Output

The analysis output consists of the following:

  • chimeras-removed.fasta: Alignment with chimeras removed
  • chimeras.removed.count_table: Count table with chimeras removed
  • chimeras-removed-summary.tsv: Summary statistics for the alignment
