Combine paired reads to contigs

Description

Combines paired end reads for each sample and then the data from all samples. Extract the sequence and quality score data from fastq files.

Parameters

<-

Details

The tool reads a forward fastq file and a reverse fastq file and outputs new fasta and report files. It extracts the sequence and quality data from the fastq files, creates a reverse complement of the reverse read and joins the reads into contigs. The algorithm aligns the pair of sequences, and then identifies possible positions where the two reads disagree. If one of the disagreeing bases has a quality score 6 or more points higher than the other, that base is reported. If not, that base is set to N. If one of the reads suggest a gap, the other read has to have a base with quality score over 25 to be considered real.
As input you give the paired end fastq files and a list of these files in .files-format. In this .files file you have 3 columns: in first column you list the sample names, in second column the forward fastq file name (read 1), and in the third column the name of the reverse fastq file (read 1). So it should look something like this:
sample1 sample1_L001_R1_001.fastq sample1_L001_R2_001.fastq
sample2 sample2_L001_R1_001.fastq sample2_L001_R2_001.fastq
sample3 sample3_L001_R1_001.fastq sample3_L001_R2_001.fastq
etc.

This tool is based on the make.contigs command of the Mothur package.

Output

The analysis output consists of the following:

  • fastq.trim.contigs.qual: Quality values for the reads
  • fastq.trim.contigs.fasta: A FASTA-formatted sequence file with the combined reads.
  • fastq-summary.tsv: Summary statistics for the sequences
  • fastq.groups: A two column list with the first column indicating the sequence names in the input file and the second column the group that it came from.
