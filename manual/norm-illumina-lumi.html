Normalisation / Illumina - lumi pipeline

Description

Normalize Illumina arrays to remove systematic bias. Please note that in order to use the lumi normalization, your raw data must be in one file. The filename must end with ".txt", and you should NOT use the Import tool for bringing the data into Chipster (in the Import files -window change the action to "Import directly").

Parameters

  • Probe identifier (Probe_ID, Array_Address_ID) [Probe_ID]
  • Transformation (none, vst, log2) [log2]
  • Background correction (none, bgAdjust.affy) [none]
  • Normalize chips (none, rsn, loess, quantile, vsn) [quantile]
  • Chiptype (empty, Human, Mouse, Rat) [empty]

Details

This tool normalizes Illumina BeadSummaryData files.

Note that you have to enter the chiptype.

If you have a BeadSummaryData that reports the probe identifiers as Array_Address_Id (for example 4900685) instead of Probe_Id (for example ILMN_2607609), change the setting for the parameter Probe identifier. Please note that processing the data with Array_Address_Id is slow and the normalization therefore takes a bit longer time than with the Probe_IDs.

You can choose to do log2 -transformation (default), vst-transformation or none. You can either skip the background correction (default: usually BeadStudio output data has already been background corrected), or use ’bgAdjust.affy’, which will call the bg.adjust function in affy package.

Within array normalization methods offer basic options: no normalization, robust spline normalization (rsn, combines features of loess and quantile normalizations), loess, quantile and variance stabilizing normalization (vsn).

Output

A tab-delimited text file containing gene names and expression estimates. This file is suitable for all further analyses.

References

This tool uses Bioconductor package lumi. Please cite the article:

Du P, Kibbe WA, Lin SM (2008) lumi: a pipeline for processing Illumina microarray, Bioinformatics, 24, 1547-1548.
