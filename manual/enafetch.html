Retrieve Data from ENA database

Description

This tool retrieves data from the ENA database based on the ID of a data set. The data retrieval and processing is done with the enaDataGet program.

Details

To be able to use this tool, you must know the ENA ID of the data set you wish to retrieve. As ENA stores data in many different levels and formats, the files retrieved form ENA may be in different formats depending on the data type that the ID refers to. In most cases you can use defaultÂ the format proposed by the enaDataGet program, but if you are retrieving NGS reads data you should explicitly define the data format to be fastq, as the SRA format, that is the default option, can't be used in Chipster. As the ENA data sets can be very large, downloading the data can take a long time.

Sample entry names:

  • ERR925793 : will return cram formatted alignment file (16030_8#1.cram)
  • ERR925793 with fastq output format : will return fastq formatted reads file (ERR925793_1.fastq.gz)
  • AH001094: will return embl formatted sequence file (AH001094.dat)
  • AH001094< with fasta output format: will return fasta formatted sequence file (AH001094.dat)
